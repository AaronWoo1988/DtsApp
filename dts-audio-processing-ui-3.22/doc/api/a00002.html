Detailed Description

Class for managing DTS accessories

AccessoryDatastore is designed to be a singleton class; the getInstance() function will return a singleton AccessoryDatastore instance that can be used to access the API functions. The getInstance() function and AccessoryDatastore class are thread-safe.

@Nullable List< QueryFilter >  queryFilters,
@NonNull QueryCallback< Brand >  callback

Queries for accessory brands based on the parameters. Only brands which have accessories will be returned; brands that do not have any accessories will be omitted. The returned brand List will be sorted in alphabetical order by brand name. The callback will be invoked from the main thread.

Parameters
contextAndroid activity or application context
queryFiltersList of filter to apply when querying. Passing an empty list or null will apply no filter and return all brands (that have accessory(ies)).
