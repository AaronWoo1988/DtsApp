Extra for indicating request ID, a unique identifier that can be used to distinguish between different intents. All requests made to the SDK (via broadcast) should include this extra to easily identify which responses correspond to which requests. final String com.dts.dtssdk.DtsIntents.INTENT_API_GET_DTS_STATE = "dts.dtssdk.intent.action.GET_DTS_STATE"
static

Intent used to indicate a request to get DTS state.

Upon receiving this intent, the SDK will do the following:

  1. Get DTS state.
  2. Broadcast an intent INTENT_STATUS_GET_DTS_RESULT containing the state of DTS and result of the request

The intent should contain an extra:

EXTRA_REQUEST_ID - A unique identifier that can be used to distinguish between different intents. and INTENT_STATUS_SET_DTS_RESULT

final String com.dts.dtssdk.DtsIntents.INTENT_STATUS_GET_DTS_RESULT = "dts.dtssdk.intent.status.GET_DTS_RESULT"
static

Intent used to indicate result for getting DTS state.

The SDK will broadcast this intent whenever it receives a request (intent) to get DTS state.

This intent will contain the following extras:

EXTRA_RESULT_CODE - Result of the get DTS request

EXTRA_DTS_STATE - State of DTS

And may contain an extra:

EXTRA_REQUEST_ID - A unique identifier that can be used to distinguish between different intents. This extra will be included if the requesting intent included it.
