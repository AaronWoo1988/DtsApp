Detailed Description

Class for retrieving DTS SDK System Status.

DtsSystemStatusChecker provides APIs to check DTS SDK states. As DtsSystemStatusChecker is designed to be a singleton class, the getInstance() function will return a singleton DtsSystemStatusChecker instance that can be used to access the API functions. The getInstance() function and DtsSystemStatusChecker class are thread-safe.

Conducts a full system status scan of all SDK components and prints out the results to a file. The function will create a new folder named "integrationtest" at the specified path, and create a new text file there with the specified name containing system status results.

If the SDK/app does not have permission to write to the specified path, it will return an error code to the specified callback (if provided).

Conducts a full system status scan of all SDK components and prints out the results to a file. The function will create a new folder named "integrationtest" at the specified path, and create a new text file there with a default name (which includes export date and time) containing system status results.

The default file name will be in the following format: DtsSystemStatusChecker-yyyy-MM-dd_HH-mm-ss.txt

If the SDK/app does not have permission to write to the specified path, it will return an error code to the specified callback (if provided).

Parameters
contextAndroid activity or application context
pathfile system path to write to. Should be application's directory if storage permission has not been granted to the app
callbackif included, the function will invoke the callback with the export results
